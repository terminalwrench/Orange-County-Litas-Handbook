# Planning Checklist

Use this checklist before approving a route for a chapter event or adding it to the route library.

## Route Readiness

- [ ] Route has been pre-ridden, recently reviewed, or validated by someone with direct local knowledge.
- [ ] Route category is accurate.
- [ ] Difficulty rating matches the full ride experience.
- [ ] Approximate distance is known.
- [ ] Estimated ride time includes realistic group pacing.
- [ ] Highway, freeway, canyon, mountain, night, or long-distance sections are identified.
- [ ] Construction, closures, and traffic patterns have been reviewed.
- [ ] Weather and seasonal conditions have been reviewed.

## Rider Fit

- [ ] Ride description is honest about difficulty.
- [ ] Beginner suitability has been reviewed.
- [ ] Technical sections are identified.
- [ ] Pace expectations are appropriate for the published ride.
- [ ] Destination-only option has been considered when useful.
- [ ] Rider count limit has been considered if the route or stops require it.

## Stops and Services

- [ ] Riders know whether to arrive fueled.
- [ ] Fuel stops are identified when needed.
- [ ] Backup fuel option exists for longer rides.
- [ ] Restroom availability is confirmed.
- [ ] Food or water access is confirmed if relevant.
- [ ] Parking capacity is appropriate for the expected group.
- [ ] Photo stops are legal, safe, and optional.

## Regroup and Communication

- [ ] Regroup locations are identified.
- [ ] Regroup locations have safe pull-off space.
- [ ] Lead and Sweep are assigned.
- [ ] Lead and Sweep both know the route.
- [ ] Lead and Sweep have each other's phone numbers.
- [ ] Pre-ride briefing points are prepared.
- [ ] Separation plan is clear.

## Contingencies

- [ ] Backup route or shortened route is available if needed.
- [ ] Weather decision timing is set.
- [ ] Plan exists for mechanical issues.
- [ ] Plan exists for a rider leaving the group.
- [ ] Alternate destination, parking, restroom, or fuel stop is identified when needed.
- [ ] Emergency response expectations are understood.

## After Approval

- [ ] Public ride description matches the approved difficulty.
- [ ] Event record links to the route or ride planning notes.
- [ ] Known hazards are included in the ride brief when relevant.
- [ ] Lead and Sweep know what should be updated after the ride.
- [ ] Lessons learned will be added to the route file or event record if the ride should be repeated.
