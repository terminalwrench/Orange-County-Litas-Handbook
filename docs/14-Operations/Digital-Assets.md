# Digital Assets

Digital assets include logos, flyers, captions, photos, videos, templates, partner graphics, and final event media.

Use this with [../03-Media](../03-Media/README.md), [../04-Social](../04-Social/README.md), and [../08-Archive](../08-Archive/README.md).

## Asset Types

| Asset Type | Where It Belongs |
| --- | --- |
| Reusable logos, fonts, icons | [../03-Media/Assets](../03-Media/Assets/README.md) |
| Flyer briefs and media templates | [../03-Media/Templates](../03-Media/Templates/README.md) |
| Event-specific final assets | Event archive or event record link |
| Social captions and recap copy | Social workflow or event record |
| Photos and videos | Media workflow and archive references |

## Operating Rules

- Keep reusable assets separate from event-specific exports.
- Preserve final versions, not every draft.
- Use consistent filenames.
- Store source files when they are needed for future edits.
- Link final assets from the event record.
- Track partner logo use and sponsor assets carefully.

## Event Asset Closeout

After each event, confirm:

- Final flyer is saved or linked.
- Final caption or event copy is preserved.
- Photo gallery or delivery link is stored.
- Video or reel assets are stored if used.
- Partner deliverables are documented.
- Reusable assets are moved to the media library if appropriate.

## Sensitive Assets

Do not broadly store:

- Private member information.
- Incident photos that should remain restricted.
- Payment information.
- Unapproved photos of people who opted out.
- Partner materials with unclear usage rights.
