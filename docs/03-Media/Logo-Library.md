# Logo Library

The logo library keeps official, chapter, partner, and event logo files organized so media contributors can find the correct file type quickly. This document does not define brand rules from The Litas headquarters. It only documents local storage, version control, and file handling.

Use this with [Asset-Library.md](Asset-Library.md).

## Approved Logo Locations

Reusable logo files belong in [Assets/Logos](Assets/Logos/README.md).

If older logo files already exist elsewhere in the repository, leave them where they are unless a cleanup project is assigned. For new Sprint 3 media work, add reusable logo files to the `03-Media/Assets/Logos/` folder and reference them from event briefs.

## Official Litas Assets

Official Litas assets are files provided by or approved through The Litas organization.

Store them with:

- Original filename, if useful for traceability
- Local standardized filename
- File type
- Source note
- Date received
- Usage approval note, if known

Do not recreate, redraw, modify, or reinterpret official assets inside this library. If a different format is needed, export from the highest-quality approved source available.

## OC Chapter Assets

OC chapter assets are files created for the Orange County chapter.

Store them with:

- Chapter identifier in the filename
- Version date
- Source file when available
- Export files used in production
- Notes about who created or approved the file when known

If an OC chapter logo has multiple versions, keep the active version easy to identify and archive older versions in a dated subfolder.

## Partner And Sponsor Logos

Partner and sponsor logos should be stored only when they are needed for an active or repeatable deliverable.

Keep:

- Source provided by partner when available
- Date received
- Event or campaign where it was used
- Contact or approval note when appropriate

Do not assume partner logos can be reused for unrelated events.

## File Types

Preferred logo file types:

- `SVG` for scalable digital use when available
- `PNG` for transparent-background digital exports
- `AI` or other editable source file when available
- `PDF` if provided as a vector source
- `JPG` only when no transparent or vector option exists

Keep the highest-quality source file. Export platform-specific versions only as needed.

## Version Control

Logo files should be versioned through filenames and repository history.

Use:

- Date-based names for received or approved versions
- Descriptive labels for source versus export files
- Separate folders for archived versions when needed

Example:

```text
oc-litas-logo-primary-source-2026-06.ai
oc-litas-logo-primary-transparent-2026-06.png
official-litas-logo-source-2026-05.svg
partner-venue-logo-transparent-2026-07.png
```

When replacing a logo, update references in active briefs and note the change in the relevant event or campaign record.
