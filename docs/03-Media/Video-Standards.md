# Video Standards

These standards cover how reels, clips, project files, exports, covers, and deliveries are organized. They do not define editing style, pacing, music choice, visual treatment, or creative direction.

Use this with [Instagram-Guidelines.md](Instagram-Guidelines.md), [Image-Sizes.md](Image-Sizes.md), [Archive-Workflow.md](Archive-Workflow.md), and [Templates/Reel-Checklist.md](Templates/Reel-Checklist.md).

## Folder Structure

For video projects, separate source material from exports.

Recommended structure:

```text
yyyy-mm-dd-event-name/
video/
  raw-clips/
  project-files/
  audio/
  captions/
  draft-exports/
  final-exports/
  covers/
```

Use only the folders needed for the project. A simple recap reel may only need `raw-clips/`, `final-exports/`, and `covers/`.

## Raw Clips

Raw clips should be kept together by event or campaign.

Track:

- Event date
- Shooter or source
- Device when useful
- External storage link if files are too large for the repository
- Usage restrictions

Do not mix raw clips from unrelated events unless creating a deliberate compilation project.

## Project Files

Project files should be stored only when they are useful for future edits or required by the production workflow.

Include:

- Editing project file
- Linked asset notes if media is stored externally
- Font, logo, or graphic dependency notes
- Export settings if they are not obvious

If a project file depends on local media paths, document where the linked media is stored.

## Draft Exports

Draft exports are for review. They should not be confused with approved final files.

Use versioned names:

```text
2026-07-18-beginner-ride-reel-v1.mp4
2026-07-18-beginner-ride-reel-v2.mp4
```

Delete or archive drafts according to the event's needs once the final export is approved.

## Final Exports

Final exports should be named clearly and stored with the event archive.

Keep:

- Final reel or video file
- Final cover image
- Caption or post copy
- Posting date
- Account tags or collaborators
- Music or audio notes when relevant
- Accessibility notes, such as captions

## Reels

For reels, confirm:

- Vertical export is ready
- Cover image is ready
- Caption is final
- Required tags are confirmed
- Collaborator invite is planned if needed
- Audio rights or platform availability are considered
- Accessibility captions are included when speech carries important information

Use [Templates/Reel-Checklist.md](Templates/Reel-Checklist.md) before publishing.

## Final Delivery

For partner, sponsor, or photographer delivery, include:

- Final file
- Delivery date
- Delivery method
- Usage note
- Credit requirements
- Contact or account receiving the file

Do not store private delivery contact details in public-facing files.
