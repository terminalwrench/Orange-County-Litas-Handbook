This module keeps Orange County Litas media assets organized and easy to use while leaving creative decisions to the designer or publishing owner.

This module does not define official Litas brand policy. It only supports local chapter operations: finding approved files, preparing photos for handoff, and checking social posts before they go live.

Use this with [Events](../02-Events/README.md), [Events Database](../10-Events-Database/README.md), and [Venues](../11-Venues/README.md) when preparing event communications.

## Assets Folder

The [Assets](Assets/README.md) folder contains reusable source assets that support media production:

- [Assets/Logos](Assets/Logos/README.md)
- [Assets/Fonts](Assets/Fonts/README.md)
- [Assets/Icons](Assets/Icons/README.md)

Final event deliverables should not be buried inside reusable asset folders. Store reusable source assets here and keep event-specific media with the related event record or leadership-approved storage location.

## Operating Standard

Every media request or publishing handoff should include:

- Event or project name
- Date and deadline
- Required deliverables
- Required logos or tags
- Exact event details
- Final file destination

Every published or delivered asset should be:

- Easy for the publishing owner to find
- Exported at the needed size
- Linked to the related event or campaign
- Credited correctly
