# Archive Workflow

The archive workflow preserves completed media so future chapter leaders can find what was published, reuse approved assets, and understand how event promotion was handled.

Use this with [File-Organization.md](File-Organization.md), [Naming-Conventions.md](Naming-Conventions.md), [Photography-Standards.md](Photography-Standards.md), and [Video-Standards.md](Video-Standards.md).

## When To Archive

Archive media after:

- Event promotion is complete
- Event recap is posted
- Final photo or video deliveries are complete
- Partner or sponsor deliverables are sent
- The event recap has been updated

For canceled or postponed events, archive the final published assets and cancellation message so future leaders can understand what happened.

## What To Archive

Keep:

- Final flyer exports
- Final caption
- Final story or reminder assets
- Final reel or video export
- Reel cover image
- Final photo selects or gallery link
- Partner or sponsor deliverables
- Source file if it is needed for future reuse
- Event brief or asset request
- Notes about platform, date posted, and accounts tagged

Do not archive:

- Duplicate drafts
- Unused exports with no future value
- Private payment records
- Personal contact details
- Sensitive incident details
- Unapproved partner materials

## Archive Folder Standard

Use event-based folders:

```text
yyyy-mm-dd-event-name/
```

Recommended contents:

```text
brief/
flyers/
instagram/
photos/
video/
partner-assets/
recap/
```

Use only the folders that apply to the event. Keep the archive readable rather than overly complex.

## Final Asset Check

Before closing the archive:

- Confirm filenames follow [Naming-Conventions.md](Naming-Conventions.md).
- Confirm final exports are clearly marked.
- Confirm source files are included only when useful.
- Confirm the event recap links to the asset location.
- Confirm partner credits and tags are captured.
- Confirm no private or sensitive information is included.

## Relationship To Event Recaps

The event recap should link to the archived media location. The archive should include enough context to understand the deliverables without reopening planning chats.

Use [../02-Events/Event-Recap.md](../02-Events/Event-Recap.md) for the event closeout process.

## Ongoing Cleanup

At least seasonally, review archived media for:

- Duplicate files
- Missing final versions
- Broken links
- Unclear file names
- Assets that should move into the reusable library
- Partner logos that are no longer approved for general reuse

Cleanup should improve findability without rewriting event history.
