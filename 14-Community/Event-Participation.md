# Event Participation

Community event participation should be intentional. OC Litas can support the broader motorcycle community without overcommitting leaders or blurring who is responsible for safety, communication, and hosting.

## Participation Levels

Use these levels when deciding how to list or discuss an external event.

| Level | Meaning |
| --- | --- |
| Calendar awareness | Leadership tracks the date to avoid conflicts. |
| Informal attendance | Members may attend independently; OC is not hosting. |
| Ride to event | OC plans a ride to an event hosted by someone else. |
| Collaboration | OC and another chapter or organization coordinate publicly. |
| Co-hosted event | OC shares planning, promotion, or day-of responsibility. |

## Decision Questions

- Is the host organization clear?
- Is OC expected to promote, ride, volunteer, donate, or attend?
- Is a route involved?
- Does the event require registration, donation, tickets, waivers, or age restrictions?
- Is the event beginner-friendly?
- Are parking, weather, traffic, and timing realistic?
- Who handles day-of communication?
- Should the event live in the Events Database as official OC participation?

## Documentation Standard

If OC plans an official ride, collaboration, or co-hosted event, create an event record in [../12-Events-Database](../12-Events-Database/). Use this module to maintain the community-event profile and long-term participation history.

Route-specific details should live in [../05-Routes](../05-Routes/) only after the route has been tested, reviewed, or completed.
