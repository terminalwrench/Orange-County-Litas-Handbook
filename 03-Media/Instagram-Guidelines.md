# Instagram Guidelines

These guidelines define the operational requirements for Instagram publishing. They do not dictate caption voice, creative style, image treatment, layout, color, or typography.

Use this with [Image-Sizes.md](Image-Sizes.md), [Naming-Conventions.md](Naming-Conventions.md), [Templates/Instagram-Post-Checklist.md](Templates/Instagram-Post-Checklist.md), and [Templates/Reel-Checklist.md](Templates/Reel-Checklist.md).

## Required Event Information

Event posts should include the information attendees need to act.

Include when applicable:

- Event name
- Date and day of week
- Start time
- Meetup time and kickstands-up time for rides
- Location or destination
- Address or RSVP instructions
- Ride status
- Beginner-friendly status
- Cost or donation requirement
- Parking note
- Weather or cancellation note
- Host, partner, or collaborator names

If an event detail is not public yet, do not imply it is final.

## Chapter Handle Usage

Use the chapter handle consistently in profile references, collaborator settings, and partner coordination. When another account posts on behalf of the chapter, confirm that the chapter is tagged or invited as a collaborator when appropriate.

Before publishing, confirm:

- Correct chapter account
- Correct event or partner accounts
- Correct venue account when tagging is approved
- No outdated or duplicate handles

## Tagging Conventions

Tags should support discovery, credit, and clarity.

Use tags for:

- Venues
- Photographers
- Partners
- Sponsors
- Collaborating chapters or groups
- Relevant event accounts

Do not tag a person, venue, or partner in a way that implies endorsement, sponsorship, or attendance if that relationship has not been confirmed.

## Collaborator Usage

Use collaborator posts when both accounts should share ownership of the post.

Confirm:

- Partner has agreed to collaborate
- Final asset is approved by all required parties
- Caption details match the shared event plan
- Posting account sends the collaborator invite before or at publication
- Someone checks that the invite was accepted

For time-sensitive events, have a backup plan if the collaborator invite is not accepted quickly.

## Accessibility Considerations

Instagram assets and captions should be understandable to as many attendees as possible.

Operational checks:

- Event details are written in the caption, not only embedded in an image.
- Images with important text have the same details repeated in the caption.
- Alt text is added when practical.
- Video content has captions or on-screen text when speech carries important information.
- Location, time, and ride details are easy to find.
- Abbreviations are clear to new attendees.

Accessibility supports attendance. It also reduces repeated questions.

## Posting Workflow

Recommended workflow:

1. Confirm event details with the event owner.
2. Confirm final asset size using [Image-Sizes.md](Image-Sizes.md).
3. Confirm required tags and collaborator accounts.
4. Review caption for accuracy.
5. Check accessibility details.
6. Publish or schedule.
7. Verify the live post.
8. Save final asset and caption according to [Archive-Workflow.md](Archive-Workflow.md).

For event changes after publishing, follow the cancellation or modification process in [../02-Events/Cancellation-Policy.md](../02-Events/Cancellation-Policy.md).
