# File Organization

This document explains where media files belong inside the OC Litas repository. The goal is to make assets easy to find while keeping reusable library files separate from event-specific deliverables.

Use this with [Asset-Library.md](Asset-Library.md), [Archive-Workflow.md](Archive-Workflow.md), and [Naming-Conventions.md](Naming-Conventions.md).

## 03-Media

Use `03-Media` for media operations standards, reusable media assets, and media request templates.

Store:

- Media workflow documentation
- Reusable logos
- Reusable font files or license notes
- Reusable icons
- Media request templates

Do not store one-time event exports in reusable asset folders.

## Assets

Use [Assets](Assets/) for reusable production assets.

| Folder | Contents |
| --- | --- |
| [Assets/Logos](Assets/Logos/) | Official, chapter, partner, and sponsor logo files that are approved for current or repeatable use. |
| [Assets/Fonts](Assets/Fonts/) | Font files or font reference notes with license information when available. |
| [Assets/Icons](Assets/Icons/) | Reusable icons or utility graphics. |

## Templates

Use [Templates](Templates/) for repeatable planning and publishing templates:

- Flyer briefs
- Instagram post checks
- Reel publishing checks
- Asset requests

Templates should remain generic enough to copy for future events.

## Event Media

Event-specific media should live with the event or campaign record, not inside the reusable asset library.

Use the event operating system in [../02-Events](../02-Events/) to connect final media to:

- Event brief
- Flyer
- Caption
- Email copy
- Route or venue notes
- Recap
- Photo or video asset link

## Flyers

Flyer briefs and final flyer exports may also be referenced from the existing flyer production area. Reusable operational requirements live in this module; active production work may remain in the existing flyer workflow.

When storing final flyer files, include:

- Feed export
- Story export
- Print export if used
- Source file when it is intended to be preserved
- Final caption or event copy link

## Photography

Photography should be organized according to [Photography-Standards.md](Photography-Standards.md).

Separate:

- Raw files
- Selects
- Edits
- Final deliveries
- Usage notes

Do not mix unreviewed camera rolls with final public deliverables.

## Video

Video should be organized according to [Video-Standards.md](Video-Standards.md).

Separate:

- Raw clips
- Project files
- Audio files
- Captions or text files
- Draft exports
- Final exports
- Cover images

## Archive

Completed media should be archived according to [Archive-Workflow.md](Archive-Workflow.md). The archive should preserve final assets and enough context to understand how they were used.

Do not archive private contact information, payment records, or sensitive incident material in general media folders.
