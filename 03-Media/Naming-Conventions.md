# Naming Conventions

Consistent names make files searchable, sortable, and understandable without opening them. Use lowercase words, hyphens between terms, and date prefixes for event-based assets.

Use this with [File-Organization.md](File-Organization.md), [Archive-Workflow.md](Archive-Workflow.md), and [Logo-Library.md](Logo-Library.md).

## General Format

Use:

```text
yyyy-mm-dd-project-or-event-asset-type-version.ext
```

Examples:

```text
2026-07-18-beginner-ride-flyer-v1.png
2026-07-18-beginner-ride-flyer-final.png
2026-08-02-beach-day-reel-cover-final.png
```

Use dates for event assets. Use descriptive names without dates only for evergreen reusable assets.

## Event Folders

Use:

```text
yyyy-mm-dd-event-name
```

Examples:

```text
2026-07-18-beginner-ride
2026-08-02-beach-day
2026-10-12-charity-ride
```

If an event date changes before publishing, use the final event date. If an event is postponed after assets were created, preserve the original record and create or rename the final folder according to the updated event plan.

## Flyers

Use:

```text
yyyy-mm-dd-event-name-flyer-format-version.ext
```

Examples:

```text
2026-07-18-beginner-ride-flyer-feed-v1.png
2026-07-18-beginner-ride-flyer-story-final.png
2026-08-02-beach-day-flyer-print-final.pdf
```

Use `final` only for files approved for publishing or delivery.

## Photos

Use:

```text
yyyy-mm-dd-event-name-photo-sequence.ext
```

Examples:

```text
2026-07-18-beginner-ride-photo-001.jpg
2026-07-18-beginner-ride-photo-002.jpg
2026-08-02-beach-day-group-photo-final.jpg
```

Raw files may keep camera filenames during editing, but selected or delivered photos should receive standardized names when archived.

## Videos

Use:

```text
yyyy-mm-dd-event-name-video-type-version.ext
```

Examples:

```text
2026-07-18-beginner-ride-reel-v1.mp4
2026-07-18-beginner-ride-reel-final.mp4
2026-08-02-beach-day-story-clip-01.mp4
```

Keep working edits separate from final exports.

## Logos

Use:

```text
organization-or-chapter-logo-description-version.ext
```

Examples:

```text
official-litas-logo-source-2026-05.svg
oc-litas-logo-primary-transparent-2026-06.png
partner-venue-logo-transparent-2026-07.png
```

Do not overwrite a logo file if the new file is materially different. Add a new version and update references.

## Version Labels

Use:

- `v1`, `v2`, `v3` for drafts
- `final` for approved deliverables
- `source` for editable originals
- `export` when a file is platform-ready but not necessarily final
- `archive` only for historical bundles or folders

Avoid names such as `final-final`, `new`, `updated`, or `use-this-one`.
